## 🏗️ 功能架构

### 前端（小程序）
- **组件位置**: `tan_sheep_wx/components/QAChat/QAChat.js`
- **功能**: 
  - 用户输入问题
  - 发送请求到后端API
  - 显示AI回答
  - 失败时降级到本地回答

### 后端（Django）
- **接口位置**: `Django_backend/sheep_management/views/qa_api.py`
- **接口路径**: `POST /api/qa/ask`
- **功能**: 
  - 接收用户问题
  - 依次尝试多个大模型API
  - 失败时使用本地关键词匹配回答

---

## 🔄 实现流程

```
用户提问
    ↓
小程序前端 (QAChat组件)
    ↓
POST /api/qa/ask { question: "..." }
    ↓
Django后端 (qa_api.py)
    ↓
尝试调用大模型（优先级顺序）:
    1. OpenAI API (GPT)
    2. 百度文心一言 API
    3. 阿里通义千问 API
    4. 本地关键词匹配（备用）
    ↓
返回回答
    ↓
小程序显示回答
```

## 🤖 大模型接入配置

### 方式一：OpenAI API（推荐）

#### 1. 获取 API Key
- 访问：https://platform.openai.com/api-keys
- 注册账号并创建 API Key

#### 2. 配置环境变量
在 Django 后端设置环境变量：

**Windows (PowerShell)**:
```powershell
$env:OPENAI_API_KEY="sk-xxxxxxxxxxxxx"
$env:OPENAI_API_BASE="https://api.openai.com/v1"
$env:OPENAI_MODEL="gpt-3.5-turbo"
```

**Linux/Mac (Bash)**:
```bash
export OPENAI_API_KEY="sk-xxxxxxxxxxxxx"
export OPENAI_API_BASE="https://api.openai.com/v1"
export OPENAI_MODEL="gpt-3.5-turbo"
```

**Django settings.py (推荐)**:
```python
import os
os.environ['OPENAI_API_KEY'] = 'sk-xxxxxxxxxxxxx'
os.environ['OPENAI_API_BASE'] = 'https://api.openai.com/v1'
os.environ['OPENAI_MODEL'] = 'gpt-3.5-turbo'
```

**或者使用 .env 文件**:
创建 `.env` 文件：
```
OPENAI_API_KEY=sk-xxxxxxxxxxxxx
OPENAI_API_BASE=https://api.openai.com/v1
OPENAI_MODEL=gpt-3.5-turbo
```

在 `settings.py` 中加载：
```python
from dotenv import load_dotenv
load_dotenv()
```

## 📝 使用说明

### 1. 默认行为（无大模型配置）

如果没有配置任何大模型API Key，系统会：
- 自动使用**本地关键词匹配**回答
- 基于预设的关键词库返回相应答案
- 提示用户"使用本地回答"

### 2. 配置大模型后

配置任一API Key后，系统会：
1. **优先使用 OpenAI**（如果配置）
2. **其次使用百度文心一言**（如果 OpenAI 失败）
3. **再次使用阿里通义千问**（如果前两者都失败）
4. **最后使用本地回答**（如果所有大模型都失败）

## 🔧 代码说明

### 后端接口 (`qa_api.py`)

```python
@csrf_exempt
@require_http_methods(["POST"])
def api_qa_ask(request):
    """智能问答接口"""
    question = data.get('question', '').strip()
    
    # 1. 尝试 OpenAI
    answer = call_openai_api(question)
    if answer:
        return JsonResponse({'code': 0, 'data': {'answer': answer}})
    
    # 2. 尝试百度
    answer = call_baidu_api(question)
    if answer:
        return JsonResponse({'code': 0, 'data': {'answer': answer}})
    
    # 3. 尝试阿里
    answer = call_ali_api(question)
    if answer:
        return JsonResponse({'code': 0, 'data': {'answer': answer}})
    
    # 4. 使用本地回答
    answer = get_local_answer(question)
    return JsonResponse({'code': 0, 'data': {'answer': answer}})
```

### 前端调用 (`QAChat.js`)

```javascript
// 调用后端大模型API
API.request('/api/qa/ask', 'POST', {
  question: question
}).then(function(res) {
  var answer = res.data.answer;
  // 显示回答
}).catch(function(error) {
  // API失败，使用本地回答作为备用
  that.handleApiError(question);
});
```

## 💡 注意事项

1. **API Key 安全**
   - ❌ 不要将 API Key 提交到代码仓库
   - ✅ 使用环境变量或 `.env` 文件（已加入 `.gitignore`）
   - ✅ 生产环境使用服务器环境变量

2. **成本控制**
   - 设置合理的 `max_tokens` 限制（当前为 1000）
   - 监控 API 调用次数和费用
   - 考虑使用缓存减少重复调用

3. **备用方案**
   - 系统已内置本地关键词匹配作为备用
   - 即使大模型不可用，也能提供基础回答

4. **系统提示词**
   - 当前已设置专业提示词，针对滩羊相关问题优化
   - 可根据需要修改 `system_prompt`

## ❓ 常见问题

**Q: 如何知道当前使用的是哪个大模型？**
A: 查看返回的 `model` 字段：`res.data.model` 会显示 `'openai'`、`'baidu'`、`'ali'` 或 `'local'`

**Q: 可以同时配置多个大模型吗？**
A: 可以，系统会按优先级依次尝试

**Q: 本地回答的关键词可以自定义吗？**
A: 可以，修改 `qa_api.py` 中的 `get_local_answer()` 函数
