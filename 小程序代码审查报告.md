# 小程序代码审查报告

## 🔴 严重问题（必须修复）

### 1. **app.json 配置错误**
**问题**: 存在重复和错误的页面路径
```json
"pages": [
    "pages/my/index",      // ✅ 正确
    "page/my/index",       // ❌ 错误：应该是 pages 不是 page
    "page/my/info-menu",   // ❌ 错误
    // ...
]
```
**影响**: 可能导致页面无法正常访问
**修复**: 删除所有 `page/` 开头的错误路径

### 2. **配置文件冲突**
**问题**: 存在多个配置文件且内容冲突
- `config.js`: `API_BASE_URL: 'http://localhost:5001/api'`
- `api-config.js`: `development: 'http://127.0.0.1:8000'`
- 同时使用了 `export default` 和 `module.exports`

**影响**: 配置混乱，难以维护
**修复**: 统一使用 `api-config.js`，删除或重构 `config.js`

### 3. **硬编码的本地地址**
**问题**: 代码中大量硬编码 localhost 地址
- `http://localhost:5001` (多处)
- `http://127.0.0.1:8000` (多处)
- `http://172.20.10.6:8080` (视频流)

**影响**: 
- 生产环境无法使用
- 无法在不同环境间切换
- 违反微信小程序规范（必须使用HTTPS）

**修复**: 
- 统一使用 `api-config.js` 管理所有API地址
- 添加环境变量支持
- 生产环境必须使用HTTPS

### 4. **HTTP协议问题**
**问题**: 多处使用HTTP协议
- 视频流地址: `http://172.20.10.6:8080`
- 图片服务器: `http://localhost:5001`
- API地址: `http://127.0.0.1:8000`

**影响**: 
- 微信小程序正式环境不支持HTTP
- 无法通过审核
- 真机测试可能失败

**修复**: 
- 开发环境可以使用HTTP（需在开发者工具中开启"不校验合法域名"）
- 生产环境必须使用HTTPS
- 配置SSL证书

## ⚠️ 重要问题（建议修复）

### 5. **项目结构混乱**
**问题**: 
- 同时存在 `page/` 和 `pages/` 目录
- 有大量压缩包文件（.zip）未清理
- 有重复的目录结构

**建议**: 
- 删除 `page/` 目录（如果不需要）
- 清理所有 `.zip` 文件
- 统一使用 `pages/` 目录

### 6. **代码质量问题**

#### 6.1 大量调试日志
**问题**: 代码中有165处 `console.log`，生产环境应该移除或使用日志级别控制

**建议**: 
```javascript
// 创建 utils/logger.js
const isDev = true; // 从配置读取
const logger = {
  log: (...args) => isDev && console.log(...args),
  error: (...args) => console.error(...args),
  warn: (...args) => console.warn(...args)
}
```

#### 6.2 错误处理不完善
**问题**: 
- 很多API调用没有try-catch
- 错误提示不够友好
- 没有统一的错误处理机制

**建议**: 
- 在 `utils/api.js` 中添加统一错误处理
- 创建错误处理中间件
- 添加用户友好的错误提示

#### 6.3 代码重复
**问题**: 
- 多个页面有相似的代码逻辑
- API请求代码重复

**建议**: 
- 提取公共逻辑到 `utils/` 目录
- 创建通用组件
- 使用混入（mixins）或组合式函数

### 7. **安全问题**

#### 7.1 API密钥硬编码
**问题**: 
```javascript
const BAIDU_MAP_AK = 'vVURIWCDRuHFMtFGxttSepBIgvm4OWGn';
```
**建议**: 
- 将密钥存储在后端
- 通过API获取密钥
- 或使用环境变量（小程序不支持，需通过后端）

#### 7.2 Token处理
**问题**: Token通过URL参数传递（不安全）
```javascript
request(`/api/cart/${cartItemId}?token=${encodeURIComponent(token)}`, 'DELETE')
```
**建议**: 
- 统一使用请求头传递Token
- 实现Token自动刷新机制
- 添加Token过期处理

### 8. **性能问题**

#### 8.1 没有数据缓存
**问题**: 每次页面加载都重新请求数据

**建议**: 
```javascript
// 添加缓存工具
const cache = {
  set(key, data, expire = 5 * 60 * 1000) {
    wx.setStorageSync(key, {
      data,
      expire: Date.now() + expire
    })
  },
  get(key) {
    const cached = wx.getStorageSync(key)
    if (cached && cached.expire > Date.now()) {
      return cached.data
    }
    return null
  }
}
```

#### 8.2 图片加载优化
**问题**: 没有图片懒加载和占位符

**建议**: 
- 使用微信小程序的 `lazy-load` 属性
- 添加图片加载失败处理
- 使用CDN加速

#### 8.3 预加载规则不合理
**问题**: 
```json
"preloadRule": {
  "pages/index/index": {
    "packages": ["feature1", "feature2", "feature3", "feature4", "feature5", "feature6"]
  }
}
```
预加载6个子包可能导致启动变慢

**建议**: 
- 只预加载用户最常用的包
- 或使用按需加载
